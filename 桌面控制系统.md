## 0. 写在前面：产品哲学

Project D 不是一个"工具"，它是一个「桌面空间体验平台」。

核心原则：
- 桌面整理是骨架 — 必须稳定、快速、零学习成本
- 壁纸和天气是皮肤 — 要有呼吸感和氛围感
- 桌宠是灵魂 — 精致、治愈、有存在感但不打扰
- 对话是心跳 — 让桌面活起来

用户在任何时候打开/关闭 Project D，桌面都应该是美的、有序的、有温度的。

## 1. 技术栈（确定，不讨论）

| 层级 | 选型 | 说明 |
|------|------|------|
| 应用框架 | Electron 28+ (Chromium内核) | 跨平台桌面应用 |
| 前端UI | Vue 3 + TypeScript + Vite | 设置面板、分区容器、对话框等所有UI |
| 壁纸渲染 | PixiJS 7+ | 2D动态壁纸、粒子特效、天气效果 |
| 桌宠动画 | Spine Runtime for Web (spine-pixi) | 精致2D骨骼动画桌宠 |
| 数据存储 | better-sqlite3 | 本地SQLite数据库 |
| 文件监控 | chokidar | 监控桌面文件变化 |
| 天气API | OpenWeatherMap (https://openweathermap.org/api) | 免费tier足够 |
| AI接口 | OpenAI Chat Completions 兼容格式 | 适配OpenAI/通义/文心/本地Ollama |
| 构建打包 | electron-builder | Windows(.exe) + macOS(.dmg) |
| 粒子引擎 | 内置于PixiJS (@pixi/particle-emitter) | 雨/雪/落叶/光斑 |

所有代码使用 TypeScript，严格模式。

```sql
CREATE TABLE IF NOT EXISTS desktop_files (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    filename        TEXT NOT NULL,                -- 原始文件名 (含扩展名)
    full_path       TEXT NOT NULL UNIQUE,          -- 完整路径
    extension       TEXT,                          -- 扩展名 (小写, 如 ".pdf")
    category        TEXT NOT NULL DEFAULT 'other', -- 分类名: program/document/image/code/archive/folder/other
    size_bytes      INTEGER,                       -- 文件大小
    modified_at     TEXT,                          -- 最后修改时间 ISO8601
    created_at      TEXT DEFAULT (datetime('now')),
    is_shortcut     INTEGER DEFAULT 0,             -- 是否为快捷方式
    custom_category TEXT,                          -- 用户自定义分类覆盖 (NULL表示使用自动分类)
    container_id    INTEGER,                       -- 所属容器ID
    sort_order      INTEGER DEFAULT 0,             -- 在容器内的排序位置
    FOREIGN KEY (container_id) REFERENCES containers(id)
);

CREATE INDEX IF NOT EXISTS idx_files_category ON desktop_files(category);
CREATE INDEX IF NOT EXISTS idx_files_container ON desktop_files(container_id);
```

### 7.1 桌面文件扫描与分类引擎

文件: `src/main/file-scanner.ts`

**输入**: 桌面路径 (`app.getPath('desktop')`)
**输出**: 分类后的文件列表 (写入数据库)

**逻辑流程**:

```
1. 获取桌面路径
   - Windows: C:\Users\{username}\Desktop
   - macOS: /Users/{username}/Desktop
   - 使用 app.getPath('desktop') 获取

2. 读取桌面目录
   - 使用 fs.readdir(path, { withFileTypes: true })
   - 获取每个文件的: name, isDirectory, isSymbolicLink

3. 过滤系统隐藏文件
   - 跳过以 '.' 开头的文件 (macOS: .DS_Store 等)
   - 跳过 desktop.ini (Windows系统文件)
   - 跳过 Thumbs.db

4. 分类逻辑
   - isDirectory → 'folder'
   - 扩展名转小写，查 EXTENSION_CATEGORY_MAP
   - 未命中 → 'other'
   - 用户有 customCategory 覆盖 → 使用覆盖值

5. 获取文件元数据
   - fs.stat() → size, mtime
   - Windows快捷方式: 检测 .lnk 扩展名 → isShortcut = true

6. 提取文件图标 (用于UI显示)
   - Windows: 使用 nativeImage.createFromPath() 或 electron-icon-extractor
   - macOS: 使用 app.getFileIcon(path)
   - 将图标保存到 userData/icons/ 缓存目录
   - 图标文件名 = 文件路径的 md5 hash + '.png'

7. 缩略图生成 (仅图片文件)
   - 使用 sharp 库将图片缩放到 200x200
   - 保存到 userData/thumbnails/
   - 懒加载: 只在容器显示时才生成

8. 写入数据库
   - INSERT OR REPLACE INTO desktop_files
   - 关联到对应的 container_id

9. 发送更新通知
   - 通过 BrowserWindow.webContents.send('desktop:files-updated', data)

10. 文件监控
    - 使用 chokidar.watch(desktopPath)
    - 监听事件: add, unlink, change
    - 增量更新数据库，不要全量重新扫描
    - 防抖: 500ms内多次变化只触发一次更新
```

**触发时机**:
- 应用启动时
- 用户点击"刷新"按钮时
- chokidar 检测到文件变化时 (防抖500ms)
- 用户点击"一键整理"时

### 7.2 桌面整理激活/归位

文件: `src/main/services/state-snapshot.ts`

**激活流程 (desktop:activate)**:

```
1. 保存当前桌面状态快照
   a. 获取桌面上所有图标位置 (平台特定)
      - Windows: 使用 COM/PowerShell 获取桌面图标位置
        命令: powershell -Command "Add-Type ...; $shell = ...; $folder = ...; $items = ...; foreach($item in $items) { ... }"
        或者使用 ffi-napi 调用 Shell32.dll
      - macOS: 使用 AppleScript / osascript
        命令: osascript -e 'tell application "Finder" to get position of every item of desktop'
   b. 将快照序列化为 JSON 保存到 app_state 表
      key: 'original_positions'
      value: JSON.stringify([{name: "file.txt", x: 100, y: 200}, ...])

2. 隐藏桌面图标 (平台特定)
   - Windows: 修改注册表 HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced, HideIcons=1, 然后通知资源管理器刷新
   - macOS: defaults write com.apple.finder CreateDesktop -bool false; killall Finder

3. 在覆盖层渲染分区容器
   - 发送 IPC 消息到渲染进程，传递分类数据和容器配置
   - 渲染进程在全屏透明窗口上绘制分区容器

4. 更新状态
   - app_state: is_active = true
   - 系统托盘图标切换到激活态
```

**归位流程 (desktop:deactivate)**:

```
1. 隐藏覆盖层
   - 发送 IPC 消息到渲染进程隐藏分区容器

2. 恢复桌面图标
   - Windows: HideIcons=0, 通知资源管理器刷新
   - macOS: defaults write com.apple.finder CreateDesktop -bool true; killall Finder

3. 恢复图标位置 (如果平台支持)
   - 从快照读取原始位置
   - 使用平台API恢复每个图标的坐标

4. 清理
   - app_state: is_active = false
   - 托盘图标切换到默认态
```

**重要**: 桌面图标隐藏/显示在 macOS 上 `killall Finder` 会导致桌面短暂闪烁，这是预期行为。在 Windows 上要尽量使用通知刷新而不是重启 explorer.exe。

### 7.4 壁纸与边框渲染

文件: `src/renderer/components/WallpaperLayer.vue` + PixiJS画布

**渲染层级**:

```
Z轴从下到上:
1. 原始系统桌面壁纸 (系统层，不覆盖)
2. PixiJS Canvas (全屏, z-index: 0)
   - 背景层: 动态壁纸/WebGL场景
   - 粒子层: 天气粒子效果
3. 分区容器层 (z-index: 10, 毛玻璃卡片)
4. 桌宠层 (z-index: 20, Spine角色)
5. UI层 (z-index: 30, 对话气泡、拉绳、指示器)
```

**壁纸切换逻辑**:

```
拉绳交互:
1. 用户点击拉绳图标 (桌面右下角，80x200px的交互区域)
2. 播放拉绳下拉动画 (CSS transform: translateY)
3. 触发壁纸切换:
   a. 检查当前壁纸索引
   b. 如果当前风格还有下一张 → index++
   c. 如果当前风格已到末尾 → 切换到下一个风格的第一张
   d. 风格循环顺序: anime → landscape → cinematic → sports → cyberpunk → minimalist → ink-wash → seasonal → anime...
4. 壁纸切换动画:
   - 当前壁纸淡出 (opacity 1→0, 0.4s ease-out)
   - 新壁纸淡入 (opacity 0→1, 0.4s ease-in)
   - 使用 PixiJS Sprite 的 alpha 属性动画
5. 播放切换音效 (可选)

壁纸加载:
- 静态壁纸: new PIXI.Sprite(PIXI.Texture.from(path))
- 适配方式: cover (保持比例, 填满, 裁切多余部分)
- 支持分辨率: 加载时按屏幕分辨率缩放，避免内存浪费
```

### 7.7 AI对话系统

文件: `src/renderer/composables/useAIChat.ts` + `src/main/services/ai-service.ts`

**API调用规格**:

```typescript
// 请求格式 (兼容 OpenAI / 通义 / 文心 / 本地 Ollama)
interface ChatRequest {
  model: string;
  messages: ChatMessage[];
  temperature: number;
  max_tokens: number;
  stream: boolean;   // 首版不使用stream，直接返回完整回复
}

// 动态构建 system prompt
function buildSystemPrompt(
  personality: PersonaDefinition,
  weather: WeatherCondition | null,
  isNight: boolean,
  currentTime: Date
): string {
  let prompt = personality.systemPrompt + '\n\n';

  // 添加当前时间上下文
  const hour = currentTime.getHours();
  let timeOfDay: string;
  if (hour >= 6 && hour < 11) timeOfDay = 'morning';
  else if (hour >= 11 && hour < 14) timeOfDay = 'noon';
  else if (hour >= 14 && hour < 18) timeOfDay = 'afternoon';
  else if (hour >= 18 && hour < 22) timeOfDay = 'evening';
  else timeOfDay = 'night';

  prompt += `当前时间: ${currentTime.toLocaleTimeString()}, ${personality.timeReactions[timeOfDay]}\n`;

  // 添加天气上下文
  if (weather && personality.weatherReactions[weather]) {
    prompt += `当前天气: ${weather}。${personality.weatherReactions[weather]}\n`;
  }

  prompt += '\n请根据以上背景，用你的人格风格与用户对话。保持简洁。';

  return prompt;
}

// 对话请求
async function chat(userMessage: string, config: AIConfig, context: {
  personality: PersonaDefinition;
  weather: WeatherCondition | null;
  isNight: boolean;
  recentHistory: ChatMessage[];   // 最近10条对话
}): Promise<string> {

  const systemPrompt = buildSystemPrompt(
    context.personality,
    context.weather,
    context.isNight,
    new Date()
  );

  const messages = [
    { role: 'system', content: systemPrompt },
    ...context.recentHistory.slice(-10),  // 保持最近10条作为上下文
    { role: 'user', content: userMessage }
  ];

  const response = await fetch(config.apiEndpoint, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${config.apiKey}`
    },
    body: JSON.stringify({
      model: config.model,
      messages,
      temperature: config.temperature,
      max_tokens: config.maxTokens,
      stream: false
    })
  });

  const data = await response.json();
  return data.choices[0].message.content;
}
```

## 12. 关键注意事项

### 平台差异

| 功能 | Windows | macOS |
|------|---------|-------|
| 桌面图标位置获取 | Shell COM接口 / PowerShell | AppleScript / osascript |
| 隐藏桌面图标 | 注册表 HideIcons | defaults write com.apple.finder CreateDesktop |
| 文件图标提取 | nativeImage + .lnk解析 | app.getFileIcon() |
| 窗口穿透点击 | 无特殊处理 (focusable:false) | 需要 NSWindow ignoresMouseEvents |
| 全屏覆盖 | 覆盖整个屏幕工作区 | 需处理多个显示器 |
| 开机自启 | app.setLoginItemSettings | app.setLoginItemSettings (mac) |
| 文件路径 | 反斜杠 `\`，需要转义 | 正斜杠 `/` |

### 性能注意

1. **文件扫描**: 桌面文件 > 500个时需要分页加载
2. **PixiJS渲染**: 粒子数控制在 300 以内，低配机器自动降低
3. **Spine动画**: 同时只渲染一个角色，切换动画时平滑过渡
4. **内存**: 壁纸图片按需加载，不要同时加载所有壁纸
5. **数据库**: 使用 WAL 模式，批量写入使用事务
6. **CPU**: 粒子系统和动画在窗口不可见时暂停渲染

### 安全

1. preload脚本使用 contextBridge，不暴露 node API
2. API Key 存储在 SQLite 中，使用 safeStorage 加密 (Electron内置)
3. 不收集任何用户数据（除非用户明确同意匿名统计）
4. AI对话内容仅本地存储，不上传到任何服务器
