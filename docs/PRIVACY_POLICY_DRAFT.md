# Project D 隐私政策（律师审核草案）

> **重要提示：本文件是产品与工程团队依据当前实现编写的工作草案，不是已生效的隐私政策，也不构成法律意见。发布、收费或开放账号功能前，必须由具备适用法域资质的律师审核，并补齐运营主体、联系方式、服务器区域、第三方清单、保存期限和未成年人规则。**

版本：Draft 0.1  
草案日期：2026-07-17  
适用产品：Project D Windows 桌面客户端

## 1. 我们是谁

Project D 的运营主体、注册地址、个人信息保护负责人和联系邮箱尚待项目负责人在正式发布前填写。当前版本属于内部测试或封闭 Beta，不应以本草案作为对外收费依据。

## 2. 核心原则

1. 桌面整理、壁纸、场景、桌宠和大部分设置在本机运行。
2. Project D 不会因为扫描桌面就自动上传桌面文件、文件内容、文件名或完整路径。
3. 天气、云端 AI、更新检查以及未来的账号与支付属于联网功能，会按本政策所述向对应服务发送必要数据。
4. 诊断报告默认在本机生成，不自动上传；导出前需要用户明确勾选同意。
5. 用户可以在隐私中心暂停外部联网服务、导出本机数据或发起彻底删除。

## 3. 当前处理的数据

| 场景 | 数据 | 处理位置与用途 | 是否默认外发 |
|---|---|---|---|
| 桌面整理 | 文件名、扩展名、完整路径、文件类型、大小、修改时间、快捷方式标记、分类和容器位置 | 在本机识别、展示、规划整理动作和恢复 | 否 |
| 门户目录 | 用户通过 Windows 原生目录选择器授权的目录名称、路径、真实路径、授权状态及目录项目元数据 | 在本机展示只读门户和搜索结果；授权可以撤销 | 否 |
| 壁纸与场景 | 本地资源路径、壁纸标识、每显示器分配、场景配置、显示器标识与布局信息 | 在本机恢复壁纸和工作场景 | 否 |
| 桌宠 | 角色、服装、人格、位置、说话频率和交互设置 | 在本机呈现桌宠 | 否 |
| AI 对话 | 用户输入、AI 回复、人格和天气上下文 | 对话历史保存在本机；启用外部 AI 时，当前输入、系统提示、天气上下文和最近最多 10 条消息发送给用户选择的 Provider | 仅在启用外部 Provider 时 |
| 天气 | 手动城市，或通过公网 IP 推断的城市、地区、国家、经纬度；天气条件、温度、湿度、风速与缓存时间 | 自动天气先向 `ipapi.co` 请求，失败时尝试 `ipwho.is`；再向 OpenWeatherMap 或 Open-Meteo 请求天气 | 自动天气开启且联网未暂停时 |
| API 配置 | 用户配置的天气或 AI API Key、Provider、接口地址、模型与参数 | Key 保存在本机；Windows `safeStorage` 可用时加密。若系统加密能力不可用，当前客户端会记录警告并可能保留兼容格式，因此不应在共享或不受信设备上保存高权限密钥 | Key 仅发送给对应 Provider；不会进入普通数据导出 |
| 本机诊断 | 应用版本、平台、架构、数据库与壁纸宿主状态、桌面项目数量、门户数量、Provider 是否配置、最多 5 条脱敏错误摘要 | 在本机生成支持摘要；路径、令牌、Key、聊天内容和原始日志不在允许字段内 | 当前不会自动上传；仅由用户主动保存导出文件 |
| 操作与恢复 | ActionPlan、执行历史、恢复状态和错误日志 | 在本机完成确认、撤销和故障恢复 | 否 |
| 账号、设备与权益 | 邮箱、显示名、设备安装标识、设备名、订单、支付渠道订单号、权益及审计记录 | Gate 5 目前只有未联网的服务端领域骨架，尚未作为正式服务启用 | 正式启用前另行告知并取得必要同意 |

Project D 不应采集桌面文件内容用于广告画像，也不应把文件名、完整路径或聊天内容加入默认遥测。

## 4. 联网服务与第三方

当前代码可能连接以下服务，实际启用情况取决于用户设置：

- IP 定位：`ipapi.co`、`ipwho.is`。请求会自然携带公网 IP，服务方可能据此返回大致位置。
- 天气：OpenWeatherMap、Open-Meteo。可能接收城市或经纬度及网络连接信息。
- AI：DeepSeek、小米 MiMo、OpenAI-compatible 自定义服务或本机 Ollama。外部 Provider 会收到第 3 节所述对话上下文。
- 更新：正式配置更新源后，更新服务器可能收到 IP、应用版本、更新通道和下载请求信息。

第三方按照其自身隐私政策处理收到的数据。正式发布前必须附上准确的服务商名称、政策链接、服务器区域和跨境传输评估；未完成这些信息时不得宣称 Gate 6 已关闭。

## 5. 处理目的与法律依据

正式政策应按实际运营地区确定法律依据。预期目的包括：

- 履行用户主动请求的桌面整理、壁纸、天气、AI 和设备权益服务；
- 保存用户设置、恢复状态并保障桌面安全；
- 在用户明确同意后生成或接收支持诊断；
- 履行订单、退款、财税和争议处理义务；
- 防止欺诈、重复回调、越权和服务滥用。

需要同意的处理不得以默认勾选、捆绑授权或拒绝非必要功能的方式取得同意。

## 6. 保存期限

当前客户端的本机数据通常保存到用户清除数据、卸载时选择删除，或操作系统账户被删除。聊天记录可在设置中清空；天气缓存约 20 分钟内优先复用，但缓存记录可能保留到被新缓存覆盖或用户删除全部数据。

正式服务端上线前必须制定并实现逐表保存期限。建议基线如下，最终期限由律师、财务和安全负责人确认：

- 活跃账号与设备：账号存续期间；解绑设备保留必要安全审计后删除或匿名化。
- 订单、支付和退款：按财税、反洗钱、消费者保护和争议处理的法定期限保存。
- 登录与安全审计：采用最短必要期限并设置自动清理。
- 已撤销权益：保留订单关联和撤销原因，以防重复开通和处理申诉。
- 诊断：只在用户主动提交后按告知期限保存；当前客户端不会自动提交。

## 7. 数据安全

客户端采用 Electron 安全 IPC、输入校验和 Windows 安全存储能力保护本机配置。Gate 5 领域骨架规定：

- 客户端不能直接把订单改成已支付；
- 支付回调必须由服务端验签并幂等处理；
- 退款会撤销对应权益并写入追加式审计；
- 客户端只缓存短期签名权益，正式版只应内置验签公钥；
- 支付私钥、管理员凭据和服务端 AI Key 不得进入客户端或代码仓库。

任何系统都无法保证绝对安全。正式上线前仍需完成渗透测试、密钥轮换、备份恢复、限流、防重放和事件响应演练。

## 8. 用户控制与权利

当前客户端提供或计划提供：

- 暂停联网服务：天气停止定位与刷新，AI 外部 Provider 停止请求并使用本地降级能力；
- 查看和撤销已授权门户；
- 导出本机用户数据，导出内容排除 API Key、访问令牌和加密秘密值；
- 清空聊天记录；
- 彻底删除本机数据并退出应用；
- 正式账号服务上线后，提供服务端数据副本、账号删除、设备解绑、更正和申诉入口。

账号删除不能违法删除依法必须保留的订单和安全审计，但应停止非必要处理并对非必要字段删除或匿名化。

## 9. 未成年人

当前产品未完成适龄设计和监护人同意流程，不应主动面向依法需要监护人同意的未成年人提供付费或云端 AI 服务。正式发布前必须由律师确定年龄阈值、监护流程、内容安全和儿童数据边界。
