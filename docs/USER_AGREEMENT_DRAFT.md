# Project D 用户协议（律师审核草案）

> **本文件是内部工作草案，不是已生效合同，不构成法律意见。正式对外发布或收费前，必须由律师审核，并补齐运营主体、适用法律、争议解决地、退款政策、联系方式和服务等级。**

版本：Draft 0.1  
草案日期：2026-07-17

## 1. 协议主体与适用范围

本协议拟用于 Project D Windows 桌面客户端及未来由同一运营主体提供的账号、设备、更新和权益服务。运营主体为 `[待填写]`。当前 Gate 5 仅有服务端领域骨架，账号和付费能力尚未正式上线；在相关能力上线并展示正式协议前，不应视为用户已经购买会员或接受自动续费。

## 2. 服务内容

Project D 可能提供桌面整理、桌面图标显示控制、静态和动态壁纸、天气效果、桌宠、AI 对话、场景、门户搜索、诊断导出、更新及未来的付费权益。具体能力以用户安装版本、设置、设备条件和页面说明为准。

产品优先保证桌面可恢复，但测试版仍可能存在兼容性问题。用户应在重要操作前保存工作并维护重要文件备份。

## 3. 桌面文件与高风险操作

1. Project D 会在本机读取桌面及用户通过 Windows 原生选择器授权的目录元数据。
2. 涉及移动、重命名、删除、覆盖或批量整理的动作应先展示计划、冲突和风险，并在需要时取得明确确认。
3. 用户不应把回收站、撤销记录或软件恢复能力当作唯一备份。
4. 软件退出、崩溃或卸载时应尽力恢复由软件隐藏的桌面图标和接管状态，但操作系统、Explorer、显卡驱动或第三方软件故障可能影响恢复。
5. 用户应确保对其整理、搜索和设置壁纸的文件拥有合法访问权。

## 4. 账号与设备

账号能力上线后，用户应提供准确、可联系的信息并妥善保护登录凭据。设备数量限制、换机解绑、异常登录和会话吊销规则应在购买前明确展示。

用户不得出售、出租、共享或自动化滥用账号，不得规避设备限制、伪造权益、篡改签名缓存或攻击服务端。发现异常后，运营方可以采取会话吊销、临时冻结和人工复核，但应提供合理申诉渠道。

## 5. AI 与第三方服务

1. Luna 的本地规则和外部 AI 输出可能不准确，不构成法律、医疗、财务或其他专业意见。
2. 涉及真实文件的高风险动作不应仅凭自然语言直接执行，仍需经过产品的确认与权限机制。
3. 用户启用外部 AI Provider 时，当前输入和有限对话上下文会发送给所选服务商；用户应阅读对应服务商条款。
4. 用户不得提交违法内容、恶意代码、他人敏感信息或其无权处理的商业秘密。
5. Provider 中断、限额或拒绝内容时，产品可能降级到本地回复，不保证每次生成成功。

## 6. 天气、定位与联网功能

自动天气使用公网 IP 推断大致城市和经纬度，并向天气服务查询。该结果可能不准确，不适用于灾害预警、航空航海或人身安全决策。用户可以使用手动天气或在隐私中心暂停联网服务。

更新、AI、天气和未来账号服务需要网络。运营方不对运营商故障、第三方 Provider 停止服务或用户网络配置导致的不可用作绝对保证，但应提供合理的错误提示和降级路径。

## 7. 付费、订单与权益

正式付费功能上线前，必须另行展示价格、币种、套餐期限、设备数量、离线宽限、退款政策和是否自动续费。除非页面明确说明并取得单独同意，Project D 不应默认开启自动续费。

商业链路应遵循：

- 客户端只创建订单和查询结果，不能自行认定支付成功；
- 支付渠道的服务端验签结果决定订单状态；
- 重复回调不得重复开通；
- 退款、拒付或被确认的风险订单可以撤销对应权益；
- 永久权益是对应产品和账号在服务规则内的长期使用权，不代表第三方服务、无限云成本或产品公司永久存续；正式销售页面必须进一步界定。

个人收款二维码或个人转账备注不得替代可审计、可验签、可退款的商业支付接口。银行卡支付应由持牌收单机构处理，Project D 不应接触或保存完整卡号、CVV 或支付密码。

## 8. 用户内容、壁纸与知识产权

用户应确保上传、导入或设置的壁纸、视频、角色素材和提示词不侵犯第三方权利。产品内置商业素材必须具有可追溯授权。对于合理的侵权通知，运营方可以暂停分发、保全证据、下架或替换内容，并向受影响方提供申诉路径。

用户保留其合法内容的权利。除实现用户主动请求所必需的本机处理外，运营方不得因本协议自动取得用户桌面文件的所有权或将其用于训练通用 AI 模型。

## 9. 禁止行为

用户不得：

- 破坏、绕过支付验签、权益签名、更新签名或设备限制；
- 利用软件传播恶意程序、违法内容或侵权资源；
- 未经授权扫描、访问、移动或删除他人文件；
- 对服务进行拒绝服务、撞库、批量盗链、回调重放或自动化刷取；
- 移除依法必须保留的版权、授权或安全标识。

## 10. 更新、暂停与终止

产品可以为修复 P0 安全或桌面恢复问题发布更新，并可暂停问题版本、在线资源、支付入口或第三方服务。远程开关不得被用于无关地访问本地文件或改变已承诺的隐私边界。

用户可以停止使用并卸载软件。卸载流程应明确选择保留或删除个人数据，并恢复由软件改变的桌面状态。账号功能上线后，用户可以按隐私政策申请删除账号；依法必须保存的订单和审计记录除外。

## 11. 责任边界

在适用法律允许范围内，测试版按当前状态提供，不承诺完全无错误或兼容所有硬件。运营方不能通过格式条款排除故意或重大过失、侵犯人身权、法定消费者权利或法律禁止排除的责任。

正式协议必须由律师根据实际销售地补充直接损失、间接损失、数据恢复、第三方服务和责任上限条款。任何责任限制都不应替代桌面恢复、支付准确和数据安全的工程义务。

## 12. 隐私

个人信息处理遵循正式发布的《Project D 隐私政策》。隐私政策与实际客户端行为不一致时，应立即停止相关处理、修复产品并依法通知，而不是仅修改文案掩盖差异。
