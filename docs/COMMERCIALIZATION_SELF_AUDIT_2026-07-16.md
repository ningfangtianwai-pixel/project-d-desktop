# Project D 商业化自查表

审查日期：2026-07-16  
审查范围：当前源码、自动化测试、最新构建/安装包、现有运行记录、当前测试电脑  
结论口径：只有“已实现且有对应实测证据”才算通过；只有代码、接口或 mock 不算商业验收完成。

## 一、结论

**发布判断：NO-GO，当前不适合直接作为付费商业正式版发布。**

- V2.1 本地客户端功能范围完成度约 **91%**，这是功能开发口径。
- 按本表加入支付后台、账号权益、内容版权、跨硬件兼容、性能基线、自动更新和长期稳定性后，商业上线准备度约 **35%**。
- 当前更适合作为 **内测版 / 封闭 Beta / 免费体验版**。
- 上线前最重要的 10 项目前为：**0 项完全关闭，2 项部分具备，8 项未完成**。

状态说明：

- `通过`：代码、真实运行和验收证据均存在。
- `部分`：有实现或局部测试，但缺关键场景或硬件证据。
- `未完成`：没有实现，或现有实现不能满足该条验收目标。
- `不适用`：当前产品未提供该能力；不得在宣传中声称支持。

### 壁纸层判断

## 十、AI 图片生成

当前 Project D 只允许 Luna 从本地壁纸库选择壁纸，不会生成新图片。因此以下能力全部为 `不适用/未建设`：

- 提示词违规拦截、图片安全审核。
- 生成队列、超时、失败重试和次数返还。
- 生成成本、存储、带宽和防刷。
- 图片质量、人脸/手部/文字检查。
- 高并发、备用模型和服务降级。

商业宣传中不得声称“AI 生成壁纸”。如果未来加入，该模块必须单独建立服务端和内容安全验收，不应直接复用当前客户端 AI 对话额度。

## 十二、风险分级清单

### P0：阻止正式发布

1. 没有真实动态视频壁纸验收和统一暂停/恢复机制。
2. 没有 Windows 登录自启动，重启后壁纸不能保证自动恢复。
3. 睡眠唤醒、最终安装包异常退出、壁纸黑屏恢复未完成闭环。
4. 没有自动更新、签名校验和失败回滚。
5. 安装包未数字签名。
6. 付费账号、订单、会员、退款、设备权益和服务端幂等均未实现。
7. 现有 entitlement 为本地开放壳，不能用于付费防篡改。
8. 壁纸授权证据链、投诉和下架机制不完整。
9. 安装器初始桌面恢复脚本与 Stage 31 运行时恢复实现不一致。

### P1：阻止大规模公开发布

1. 无 CPU/GPU/帧率标准和 4 小时/24 小时稳定性曲线。
2. 全屏游戏和电池状态没有真正控制壁纸暂停/降级。
3. 不支持每屏独立壁纸。
4. Windows 10、AMD、NVIDIA、2K/4K、双/三屏、不同刷新率未验证。
5. DPI 100/125/150/200%、竖屏、热插拔缺物理证据。
6. 没有干净账号、受限账号和杀毒软件矩阵。
7. 无正式隐私政策和第三方安全审计。

### P2/P3

- P2：壁纸裁剪策略、错误提示、下载/导入流程、操作反馈和可访问性。
- P3：文案、动画节奏、缩略图一致性和细节视觉优化。

P2/P3 不应早于上述 P0/P1 被作为主要商业化工作。

## 十三、建议的关闭顺序

1. **Stage 32：壁纸可靠性与性能**  
   增加真实视频资产、统一暂停状态机、全屏/电池自动策略、开机自启动、安装器恢复脚本统一、CPU/GPU/内存采样工具。
2. **Stage 33：硬件矩阵与长测**  
   完成多屏独立映射、4K/竖屏/DPI、睡眠唤醒、热插拔、Windows 10/11、Intel/AMD/NVIDIA、4小时/24小时测试。
3. **Stage 34：更新、账号与支付服务端**  
   签名安装包、签名更新、失败回滚、账号设备、订单幂等、支付回调、权益恢复、退款撤销。
4. **Stage 35：内容与合规**  
   建立授权档案、许可快照、作者协议、侵权投诉、下架流程、隐私政策和安全审计。

完成 Stage 32 和 Stage 33 后，可考虑公开免费 Beta；Stage 34 和 Stage 35 未完成前，不建议以会员制付费软件正式销售。
